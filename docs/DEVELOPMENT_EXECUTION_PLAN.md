## 1. 执行规则

- 状态只使用：`未开始`、`执行中`、`待验收`、`已完成`、`外部阻塞`、`明确延期`。
- “已完成”必须同时具备实现、自动化测试、文档和 `npm run release:check` 证据，不能只凭代码存在。
- 涉及数据库事务的项目必须同时通过 SQLite 与 PostgreSQL contract 测试。
- 涉及真实 Provider、真实 Code 客户端或跨机器协作的项目，在自动化测试通过后保持“待验收”，
  直到脱敏验收记录归档。
- 每次状态变化都在本文“执行记录”中追加日期、分支、SHA/工作树状态、测试结果和遗留风险。
- 未完成 N006–N008 前，Gateway 不得暴露到非回环地址；未完成 N010–N012 前，积分只作为
  开发数据，不得用于真实计费或限额承诺。
